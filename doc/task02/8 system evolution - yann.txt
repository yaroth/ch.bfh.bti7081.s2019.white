System evolution

Die Systemevolution beschreibt die grundlegenden Annahmen, auf denen das System basiert, und alle erwarteten Änderungen aufgrund von Hardwareevolutionen, sich ändernden Benutzeranforderungen usw. Dieser Abschnitt ist für Systementwickler nützlich, da er ihnen helfen kann, Designentscheidungen zu vermeiden, die mögliche zukünftige Änderungen am System einschränken würden.

Annahmen:
- Programmiersprache: Java.
- Fürs grafische Interface: Vaadin.
- Client-Server Applikation: Daten werden auf einem zentralen Server gespeichert.
- Datenschutz: eine sichere Verbindung zwischen dem Client und dem Server muss gewährleistet sein.
- Die Applikation muss mobile tauglich sein: Webapp oder Native App.


Mögliche Änderungen (in Zukunft):
- Datenschutz kann in Zukunft erleichtert oder erschwert werden, je nach Bestimmungen.
- Applikation ist zur Zeit nur für ein einziges Gerät konzipiert. Könnte in Zukunft auf mehrere Geräte ausgeweitet werden.(Fitnessband, Armbanduhr, VR Brille).
- Neben Stimmung, Tips und Kalendereinträge könnten in Zukunft weitere Themen hinzukommen.
- Mobile Native App entwickeln. Dies würde es ermöglichen Offline zu arbeiten.
